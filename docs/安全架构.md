# 安全架构

## 认证

- JWT 认证通过 `JWT_SECRET` 加载密钥；缺失时 API 启动失败。
- 浏览器 access token 默认有效期为 30 分钟。
- 浏览器 refresh token 存放于 HttpOnly Cookie，连续 7 天无刷新后失效。
- 浏览器 refresh token 只存储 SHA-256 哈希，刷新成功时滑动延长 7 天。
- 密码和 Agent Secret Key 使用 bcrypt cost 12 存储。
- 登录时对不存在用户执行固定 bcrypt 比较，避免通过耗时差异枚举用户名。
- 公开路由使用 `@Public()` 标记；公开路由上的无效 JWT 会降级为匿名访问。
- 用户退出登录时撤销当前浏览器会话，并清除 refresh Cookie。
- token version 保留为全局失效机制，用于旧 token 兼容、封禁和后续“退出全部设备”等场景。

## 速率限制

- 全局限流使用 Redis 存储。
- 当前全局限流连接 Docker 网络内的 `redis:6379`。
- 全局 short 限流：1 秒 10 次。
- 全局 medium 限流：10 秒 50 次。
- 全局 long 限流：60 秒 300 次。
- 注册接口限流：60 秒 3 次、3600 秒 10 次。
- 登录接口限流：10 秒 5 次、60 秒 15 次。

## API 响应

- 全局 `ValidationPipe` 开启 `whitelist`、`forbidNonWhitelisted`、`transform`。
- 全局响应拦截器会把普通返回值包装为 `{ data: ... }`。
- 已包含 `data` 字段的响应会直接透传。
- HTTP 异常统一返回 `{ error: { code, message, statusCode } }`。
